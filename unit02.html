Scenario

You are setting up as an independent consultant, which is very exciting, however, you’re going to need a website of your own to highlight your skills, availability and areas of work. It’s time for you to create a website that details you, and including pages/sections and a how you built it page:

  • 1.1 Describing the content and layout for each page

    This design is divided in 4 distinct pages:

    • Home : This page contains information relating to the design of the site i.e. requirements
    • Unit 1 Outcome : This page contains information relating to the Unit 1 tasks, each task contains readable text that becomes visible when the area is expanded using the “+” button.
    • Unit 2 Outcome : This page contains information relating to the Unit 2 tasks, each task contains readable text that becomes visible when the area is expanded using the “+” button.
    • About : The about page contains basic information about the author (similar to a mini CV) and also the capability to follow social networking links to the author.

    In addition some content is available across all or some pages (by design):

    • All pages : Navigation bar, footer, header graphics
    • Outcome pages (1 and 2) : Information bar that lists tools and websites with code examples

    Shared across all pages is a common style sheet – there is no local styling within the HTML

    Also shared across all pages is a reference to a JavaScript file that contains all of the required functions, however, each page has a pageInitialise() function (called within body:onload()) that only calls in the functions if they are actually required.

  • 1.6 Describing access issues that need taking into account - describe issues that people with disabilities may face when using webpages and how we can overcome them

    Some considerations to take into account when designing a website that could be accessed by an audience that has any type of disability (non-disabled users also benefit in some cases):

    • Use alt tags - for someone who has a visual impairment and uses a screen reader the alt tags are read aloud
    • Describe links - when embedding a link, it's more useful to describe the link, rather than just telling the reader to "click here" this also helps with screen readers
    • Subtitles and transcripts - if web content includes videos, try to provide subtitles
    • Write abbreviations in full - for example "B.Y.O.D." rather than "BYOD"
    • Colours - smart color choice is necessary, avoid pairing garish colors, and be wary of using yellow, blue and green in close proximity as this is especially difficult for colorblind users
    • Increased size click areas - to help users with mobility issues, within reason provide them a wider click range to make things more accessible
    • Keep it simple - for elderly audiences or those with learning disabilities break out text into smaller paragraphs and use simple, straightforward language
    • Increased font size - easier to read for someone who has a visual impairment

  • 1.7 Describe which are the appropriate filetypes for websites

    Various type of files are commonly found on the internet or example:

    • AVI — Audio/Video Interleaved
    • CSS — Cascading StyleSheet
    • DOC — Microsoft Word DOCument
    • EXE — EXEcutable file
    • GIF — Graphics Interchange Format
    • HTM — HyperText Markup Language file
    • HTML — HyperText Markup Language file
    • JPG — Joint Photographic Experts Group file
    • JPEG — Joint Photographic Experts Group file
    • MID — Musical Instrument Digital Interface
    • MIDI — Musical Instrument Digital Interface
    • MP3 — MPEG Layer 3 sound file
    • MPG — Motion Picture Experts Group file
    • MPEG — Motion Picture Experts Group file
    • MOV/QT — QuickTime MOVie
    • PDF — Portable Document Format
    • PNG — Portable Network Graphics
    • RAM — Real Audio Movie
    • RAR — RAR archive
    • TIFF — Tagged Image File Format
    • TXT — Text file
    • WAV — WAVe sound file
    • ZIP — ZIPped file

    Whilst all of the above files (and more) are present on the internet, designers should be aware that filtering should be applied to prevent viruses and malware; for example when using an upload control only allow specific expected types of files to reduce potential exposure.

  • 1.8 Storing and retrieving files effectively

    It is important that the website code and any uploaded files are kept separate. In an ideal world a security strategy should be in place so that up loadable and changeable files are stored in a read/write area of storage with clear separation from that of the website static code and content that should only have read access.
